# ImonEngine Store Autopilot Roadmap

## End State

The program ends only when all of the following are true:

- The Gumroad store has at least 5 polished products, with the first 2 already live or upload-ready and the rest generated in a repeatable pipeline.
- Product creation is documented and automated enough that new digital assets can be produced without manual design work.
- Sales workflows exist for store conversion, email capture, affiliate/discover readiness, and lightweight traffic generation using free or no-cost channels.
- ImonEngine can track the store lane cleanly on the VPS, with updated docs, commands, and resumable handoffs.
- The final phase sends a completion email to `joshuabigaud@gmail.com` from the signed-in ImonEngine browser session.

## Broad Plan

1. Create an automated product creation workflow.
2. Expand the store catalog to 5 total products.
3. Automate the first sales and growth loops.
4. Harden the operating system around the store so ImonEngine can run it smoothly on the VPS.
5. Close the loop with documentation, state handoff, and final notification.

## Ordered Phases

### Phase 1: Product Factory Expansion

- Finish the second digital asset and publish it if browser/file constraints allow.
- Build 3 additional products for the Gumroad store.
- Reuse or improve repo scripts so product generation becomes repeatable instead of manual.
- Keep all work inside the existing digital asset business lane instead of opening a new business type.

### Phase 2: Store Conversion Automation

- Improve storefront sections, copy, tags, thumbnails, and product organization.
- Create at least one free or low-friction entry offer, sample, or lead-capture path if it can be done without spend.
- Configure Gumroad-native workflows or follow-up flows that do not require paid tools.
- Document what is live, what is deferred, and what metrics matter.

### Phase 3: Growth Automation

- Create no-cost traffic workflows using free channels, free tiers, or owned channels.
- Reuse products to generate content assets, previews, snippets, or lead magnets.
- Set up lightweight publishing or drafting workflows that can keep producing sales opportunities.
- Prefer free platforms and free plans. Do not spend money.

### Phase 4: Autonomous Operations Hardening

- Improve ImonEngine scripts, docs, and VPS workflows for the store lane.
- Add or refine commands, cron-safe flows, and handoff logs needed for low-touch operations.
- Make sure GitHub remains the source of truth for code while the VPS remains the source of truth for secrets.
- Leave the browser session open whenever possible to preserve account access.

### Phase 5: Final Review And Notification

- Confirm the roadmap end state has been met or document any remaining blockers.
- Update the roadmap, handoff docs, and store docs with final status.
- Commit, push, and sync the final changes.
- Use the signed-in browser session to send a completion email to `joshuabigaud@gmail.com`.

## Global Constraints

- No spending money.
- Free tier signups are allowed.
- Use the signed-in ImonEngine browser profile/session when available.
- Use the VPS and OpenClaw access where it helps automation, but keep secrets on the VPS.
- Prefer automated or scripted workflows over one-off manual steps.
- Do not wait for human input unless blocked by something that cannot be bypassed safely.
